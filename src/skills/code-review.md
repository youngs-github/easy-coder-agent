# 代码审查

你正在执行代码审查任务。请按以下流程操作：

## 审查步骤

1. 使用 Read 工具读取待审查的文件
2. 逐项检查以下方面：
   - **正确性**: 逻辑是否正确，边界情况是否处理
   - **安全性**: 是否有注入风险、敏感信息泄露
   - **性能**: 是否有明显的性能问题（N+1 查询、不必要的循环等）
   - **可维护性**: 命名是否清晰、函数是否过长、注释是否充分
   - **错误处理**: 错误是否被正确捕获和处理
3. 按严重程度分类输出结果

## 输出格式

```
## 审查结果

### 🔴 必须修复
- [文件:行号] 问题描述

### 🟡 建议改进
- [文件:行号] 问题描述

### 🟢 亮点
- 值得肯定的做法
```

## 注意事项

- 不要只关注负面问题，也要指出写得好的地方
- 给出具体的修改建议，不要只说"有问题"
- 区分风格偏好和实际问题
